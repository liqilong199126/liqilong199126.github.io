Vue项目构建总结

发表于 2017-07-04 | 分类于 vue.js

一.构建vue项目的分类

1.直接引入vue.js文件

  • 全局定义组件，只能使用一些基础的API和局限的功能，不能使用预处理器，字符串模板缺乏语法高亮…
  • 一般主要用于初级用户和小型项目

2.使用vue单文件组件

  • 完整语法高亮, CommonJS模块,组件化的CSS
  • 适用于大型项目

二.构建工具与方法

1.构建工具

  • webpack
  • browserify

2.构建方法

  • vue-cli脚手架
  • 自定义构建

三.vue-cli构建

但因为国内网络原因，使用 npm 安装某些模块会非常慢。好在淘宝提供了 npm 镜像供我们使用。

1.安装 cnpm

在命令行输入：

npm install -g cnpm –registry=https://registry.npm.taobao.org

2.使用 cnpm 安装模块

cnpm install [name]

如果你使用vue-cli脚手架来构建vue项目，那么你只需敲击下面几行命令即可生成一个简单的vue项目（前提安装node.js）：
安装vue-cli之前，需要先装好vue 和 webpack

npm install -g vue //全局安装vue 也可省略

npm install -g webpack //全局安装webpack

npm install -g vue-cli //全局安装vue-cli

注意：上面这些装过一次之后都不需要再安装了

3.使用vue-cli构建vue项目

vue init webpack my-project

cd my-project

npm install

npm run dev

5.vue-cli开发环境 关于Cannot GET /的问题

　　针对开发环境需要注意，一定不要去改asssetsPublicPath里面的内容，也就是这个地方是不要加点 改成当前目录的；否则会出Cannot GET 找不到页面的问题；

这样的构建方式其实并不适用于所有项目，很多文件你的项目可能都不会用到，并且如果你对自动生成的文件一无所知，那么后期维护起来也会非常的吃力。所以这里不推荐新手使用vue-cli构建，而是推荐大家参考vue-cli生成的文件从零开始构建一个vue项目。

四.自定义构建

相比vue-cli构建，自定义构建就显得灵活得多，但是它需要你了解构建的步骤和原理，要求也就随之提高了。自定义构建分为以下几步：

  • 文件/文件夹创建
  • package.json文件创建
  • webpack配置文件创建
  • 入口文件创建
  • vue组件编写
  • 路由配置
