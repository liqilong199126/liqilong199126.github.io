webpack学习笔记

发表于 2017-07-15 | 分类于 webpack

webpack基本介绍

1.webpack基本介绍

2.webpack的安装和命令行

  1. 在相应文件夹下：npm install webpack –save-dev

  2. 打包hello.js文件：webpack hello.js hello.bundle.js

（hello.bundle.js为打包后的文件）

  • Asset: 打包生成的文件
  • Size:文件大小
  • Chunks:打包的分块
  • Chunk Names:打包的块名称
  1. webpack天生不支持css文件，需要安装loader：
    npm install css-loader style-loader –save-dev

  1. css-loader使得webpack可以处理.css文件，style-loader通过css-loader处理完的文件，把处理完的文件新建一个style标签插入到html文档中，当html引入打包后的文件后，改代码会被执行。

  2. 可以不需要每次在require指定css-loader和style-loader

从而使用module-bind参数

webpack hello.js hello.bundle.js –module-bind ‘css=style-loader!css-loader’

6.
(1)文件自动改变后自动打包 使用watch参数

webpack hello.js hello.bundle.js –module-bind ‘css=style-loader!css-loader’ –watch

(2)看到打包过程 –progress参数

webpack hello.js hello.bundle.js –module-bind ‘css=style-loader!css-loader’ –progress

(3)看到打包的模块 –display-modules参数

webpack hello.js hello.bundle.js –module-bind ‘css=style-loader!css-loader’ –display-modules

(4)看到打包的原因 –display-reaons参数

webpack hello.js hello.bundle.js –module-bind ‘css=style-loader!css-loader’ –display-reasons

挖坑填坑2

2.0版本直接合并会报错“Conflict: Multiple assets emit to the same filename bundle.js
”，但是文件依然会生成。官方给出的标准是两个文件打包后还是两个文件，并不进行合并。


解决：filename修改如下

filename区别

  1. [name].js
    生成的是源文件名称的两个文件夹

  2. [name] - [hash].js
    打包后的文件名就是源文件名加上hash值

源Hash: 6ed8fcb869f32b6d0cf6

生成的是main-6ed8fcb869f32b6d0cf6.js

注意的是 这个hash值和文件名hash值相对

  1. [name] -[chunkhash].js

和上面一样 生成的文件名就是源文件名加上hash

唯一不同的是

文件名的hash值和生成的hash值不一样

源Hash: 18a8a031019648cb4892

生成的是main-739dc04ca19361abc6a7.js

挖坑填坑3

webpack版本1.X与2.X有很大不同，

  1. 版本更新的话，Loader可以改成loader: ‘babel-loader’

即在webpack.config.js里面，向loader传参不在使用query，而是使用options
{test: ‘/.js$/‘, loader: ‘babel-loader’, options: {‘presets’: [‘env’]}} （-loader不可以省略）

  1. 另外如果是在.babelrc文件中配置的话，千万不要给.babelrc文件添加诸如js, json的后缀名，这个文件没有后缀，切记！切记！

  2. 安装方法 npm install babel-loader babel-core babel-preset-env webpack –save-dev

  3. 所以后面的参数’由presets: [‘latest’]相对于的变成了presets: [‘env’]

  4. 官方并没有废弃query 也没有指明options是新参数 实际测试两种方法都可以 生成结果也一模一样

  5. loader: ‘babel-loader’才能被识别

  6. include和exclude需要相对路径，所以include:dirname +’./src/‘ 加前缀dirname

  7. babel-loader官网表示 { “presets”: [“latest”] } === { “presets”: [“env”] }
